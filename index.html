AirSim仿真环境搭建

AirSim环境及接口配置（双目图像+ground truth）

  • 环境配置

    • Windows 10
    • AirSim v1.6.0
    • Anaconda python 3.7+

  • AirSim

    • Doc：https://microsoft.github.io/AirSim/
    • Github: https://github.com/microsoft/AirSim
    • Environment: https://github.com/Microsoft/AirSim/releases

  • 使用

    • 从Release中下载编译好的UE4环境，如Blocks.correct.zip，并解压（PS：请尽量将环境解压到固态硬盘上，以获得更好的io速度）
    • 在解压位置\Blocks.correct\Blocks\WindowsNoEditor中找到Blocks.exe双击运行
    • 初次运行可能需要安装DirectX Runtime，可以从这里下载
    • 初次运行时可以选择Car或者Multirotor，后续可以在Setting中进行修改指定一种
    • Press F1可以看到设置，Alt+Enter可以切换全屏/窗口模式
    • 已知问题: 目前1.4.0版本中的AbandonedPark环境会出现报错，待查明，可以先体验Blocks环境

  • Setting

    • ref1: https://microsoft.github.io/AirSim/settings/
    • ref2: https://github.com/Microsoft/AirSim/blob/master/docs/settings.md
    • 位置：C:\Users\XXX\Documents\AirSim, XXX为用户名

  • 体验

    • 安装airsim python包：pip install airsim， 显示Successfully installed airsim-1.3.0 msgpack-python-0.5.6 msgpack-rpc-python-0.4.1 opencv-contrib-python-4.5.1.48 tornado-4.5.3
    • 运行Blocks.exe
    • 运行hello_drone.py
    • 其他示例代码：https://github.com/microsoft/AirSim/tree/master/PythonClient/multirotor

  • 注意

    • 有时候会出现连接不上的情况，需要关掉UE4环境，并从任务管理器中结束所有UE4的进程，再重新打开环境进行连接

  • 双目

    • related issues:
      • https://github.com/microsoft/AirSim/issues/1770
      • https://github.com/microsoft/AirSim/issues/2830
    • 修改AirSim中的图像显示
      • 在Setting中加入SubWindows:
      • 结果如下：
    • Camera ID
      • front_center, front_right, front_left, bottom_center and back_center分别对应0, 1, 2, 3, 4
    • Image Type
      • Scene = 0, DepthPlanar = 1, DepthPerspective = 2, DepthVis = 3, DisparityNormalized = 4, Segmentation = 5, SurfaceNormals = 6, Infrared = 7
    • 修改相机基线长度或者分辨率
      • 可以直接在setting中进行设置，参考这个回答。该回答也顺便解决了相机配置问题，比如分辨率什么的。
    • 在python中获取左右图片和深度信息
      • 无人机模式
        • 在settings.json中确认SimMode为Multirotor模式
        • 使用scripts/stereo_cam.py获得图像
      • CV模式
        • 在settings.json中设置SimMode为ComputerVision模式
        • 使用scripts/stereo_cam_cv_mode.py获得图像
        • CV模式操作：
          • 箭头控制前后左右
          • Page Up/Down 控制上下
          • W,S控制pitch
          • A,D控制yaw
      • 存在问题
        • 获取深度图像速度慢，在默认分辨率（256*144）情况下只有5Hz（GTX1060 6G）
        • 降低深度图像分辨率可以变快，但是已经挺低的了目前
        • 获取图像时设置pixels_as_float = False 之后虽然速度可以到20+fps，但是深度图像无法解析
        • 相关issue：
          • https://github.com/microsoft/AirSim/issues/1761

GitHub加速方法

本文旨在解决git clone ，直接下载ZIP文件，以及Release中的asset文件速度慢的问题。

1. GitHub镜像访问

这里提供一个常用的镜像地址：

  • https://hub.fastgit.org

此网站是 GitHub 的镜像，适合在 GitHub 也无法登陆的时候可以使用。可以在此进行下载克隆等操作。

2. 油猴插件

油猴借助网页的scripts，能够实现对网页的定制化修改。能够实现诸如去广告，自动完成验证码，免登录看视频，跳过视频片头，解析视频链接，下载视频等等诸多功能。这里列举的只是九牛一毛。使用油猴加速的原理是将下载(clone)的地址修改为镜像的地址，从而实现加速。

2.1插件安装

首先要给浏览器安装油猴插件

2.1.1 Chrome

Chrome插件有两种方式可以下载

  • 从商城中下载。（需要科学上网）
  • 进入开发者模式，从内网下载插件后自己安装。

2.1.2 360极速浏览器

在地址栏中输入 https://ext.chrome.360.cn/ 搜索暴力猴ViolentMonkey安装即可。

2.1.3 Edge浏览器

在地址栏中输入 https://microsoftedge.microsoft.com/addons/Microsoft-Edge-Extensions-Home 搜索 Tampermonkey 安装即可。

2.2 使用加速脚本

https://greasyfork.org 是一个油猴脚本的发布站，里面有许多奇奇怪怪的脚本可用。这里我们直接使用 https://greasyfork.org/scripts/412245 脚本下载安装即可。

3 构造链接下载

在要下载的路径前加 https://gh.api.99988866.xyz/ 即可。
例如要下载 https://github.com/xxxx/archive/master.zip 文件，只需构造新的URL:https://gh.api.99988866.xyz/https://github.com/xxxx/archive/master.zip ，输入到地址栏中即可开始下载。

4 Gitee

直接在Gitee中搜索下载。如果Gitee中没有，需要自己新建仓库从GitHub中导入。
这种方法比较麻烦，而且实时性不好。

5 走代理 & 改hosts

以上方法均不解决clone时带有submodule的仓库。若是需要clone带submodule的仓库要么开代理，要么改hosts文件。
可以在电脑上或者路由器上配置代理，从而实现走代理下载。
一劳永逸，就是比较麻烦，有时还需要RMB。
